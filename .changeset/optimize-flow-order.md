优化配置流程顺序：连接方式优先，同屏展开体验

**改进内容**：

1. **连接方式优先**：
   - 现在首先询问连接方式（官方服务/自定义中转站）
   - 符合"目标优先于凭据"的用户心智模型
   - 与行业标准工具（数据库客户端、SSH 工具）一致

2. **同屏展开效果**：
   - 选择"自定义中转站"后立即询问 API 地址
   - 无需等待下一步，流程更加流畅
   - 减少认知负荷和操作步骤

3. **配置名称后置**：
   - 配置名称移到最后一步
   - 根据连接方式自动生成默认值（"官方"或"自定义中转"）
   - 用户可以直接使用默认值或自定义

4. **显式选择代替隐式规则**：
   - 移除"空值=官方"的隐式语义
   - 通过选择器显式表达用户意图
   - 避免字段留空的歧义和错误

**新流程**：
```
创建配置：
1. 连接方式 → (如果自定义) API 地址
2. API Key
3. 其他配置（代理、禁用流量）
4. 配置名称（自动生成默认值）

编辑配置：
1. 选择配置
2. 连接方式 → (如果自定义) API 地址
3. API Key（可选）
4. 其他配置
```

**用户体验提升**：
- 更符合直觉的流程顺序
- 减少了用户的思考时间
- 避免了字段语义歧义
- 配置名称智能默认，降低命名负担
